# Security Policy

We take the security of this project, its users, and its data very seriously. If you believe you have found a security vulnerability in Skills Transmuter, please report it to us responsibly so that we can address it as quickly as possible.

---

## Supported Versions

Currently, security updates are actively provided for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | :white_check_mark: |
| < 1.0.0 | :x:                |

---

## Reporting a Vulnerability

**Please do not report security vulnerabilities via public GitHub issues.**

If you discover a vulnerability, please report it privately by opening a secure, private draft advisory on the GitHub repository or by contacting the repository maintainers directly.

### What to Include
To help us triage and resolve the issue quickly, please include:
* A detailed description of the vulnerability and its potential impact.
* Clear steps to reproduce the issue (including proof-of-concept markdown skills or command flags if applicable).
* Details about the operating system and Node.js environment where it was observed.

---

## Our Response Process

1. **Acknowledgment:** We will acknowledge receipt of your report within **48 hours** and coordinate a resolution timeline.
2. **Investigation:** We will investigate the issue and, if verified, develop a patch.
3. **Disclosure:** Once a patch is released, we will publish a security advisory or coordinate public disclosure with you, giving full credit for your responsible disclosure.

We ask that you do not disclose the vulnerability publicly or to any third party until a patch has been made available and applied.

Thank you for helping keep the agentic AI community and its tools safe!
